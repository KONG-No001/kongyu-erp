# 账号与权限接口设计

## 1. 文档目的

本文档用于定义账号与权限模块对外提供的接口规范，包括：

* 账户管理接口
* 角色管理接口
* 权限资源接口
* 权限查询接口（前端使用）
* 鉴权辅助接口

用于指导：

* 后端 Controller 开发
* 前后端联调
* 接口测试与Mock

**本文档仅定义接口协议与交互规范，不涉及具体实现细节，具体实现由开发阶段完成。**

---

## 2. 设计原则

* API规范：统一采用 `/api/{domain}/{resource}` 结构，遵循 RESTful 风格，部分复杂操作通过 action 扩展路径实现。
* 返回结构统一
* 权限控制基于 `access_resource`
* 接口鉴权统一通过 Spring Security
* 所有写操作具备幂等性设计（结合数据库约束）

---

## 3. 通用约定

### 3.1 API格式

所有接口统一以 `/api` 作为前缀。

API 路径规范参照《通用基础设施组件设计》文档中的 API 规范：

* 统一采用 `/api/{domain}/{resource}` 结构
* 遵循 RESTful 风格（资源 + HTTP Method）

在以下场景允许使用 action 后缀扩展：

- 分页查询（`/page`）
- 状态变更（`/status`）
- 批量操作（`/batch/*`）

### 3.2 返回结构

返回结构参照《通用基础设施组件设计》文档中的 Result 规范：

* 所有接口统一返回 `Result<T>`

### 3.3 分页查询

分页接口设计参照《通用基础设施组件设计》文档中的分页规范：

* 统一采用 **POST**，支持复杂查询条件
* 统一采用 `/page` 作为 Path 后缀
* 请求参数统一使用 `PageRequest.class`
* 返回结果统一使用 `Result<PageResult<T>>`

# 5. 角色管理接口

---

## 5.1 查询角色列表

**GET** `/api/roles`

---

## 5.2 新增角色

**POST** `/api/roles`

---

## 5.3 修改角色

**PUT** `/api/roles/{id}`

---

## 5.4 删除角色（逻辑删除）

**DELETE** `/api/roles/{id}`

---

## 5.5 绑定权限

**POST** `/api/roles/{id}/permissions`

```json
{
  "permissionIds": [
    1,
    2,
    3
  ]
}
```

---

## 5.6 解绑权限

**DELETE** `/api/roles/{id}/permissions`

# 7. 账户权限（直绑）接口

---

## 7.1 设置账户权限（ALLOW / DENY）

**POST** `/api/accounts/{id}/permissions`

```json
{
  "permissions": [
    {
      "permissionId": 10,
      "effectType": "ALLOW"
    },
    {
      "permissionId": 11,
      "effectType": "DENY"
    }
  ]
}
```

---

## 7.2 取消账户权限

**DELETE** `/api/accounts/{id}/permissions`

# 8. 权限查询接口（前端使用）

---

## 8.1 获取当前用户权限集合

**GET** `/api/me/permissions`

---

### 返回示例

```json
{
  "permissions": [
    "sys:account:add",
    "btn:sys:account:add",
    "menu:sys:account",
    "data:sys:account:show-phone"
  ]
}
```

---

## 8.2 获取菜单树

**GET** `/api/me/menus`

# 9. 鉴权约定

---

## 9.1 API 权限

后端接口需声明：

```java
@PreAuthorize("hasPermission('sys:account:add')")
```

---

## 9.2 按钮权限

前端根据：

```json
"btn:sys:account:add"
```

控制显示

---

## 9.3 DATA / FILE / CAP

在业务代码中显式判断：

```java
if(hasPermission("data:sys:account:show-phone")){
        // 展示手机号
        }
```
