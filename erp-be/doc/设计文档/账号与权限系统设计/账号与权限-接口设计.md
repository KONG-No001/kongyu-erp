# 账号与权限接口设计

## 1. 文档目的

本文档用于定义账号与权限模块对外提供的接口规范，包括：

* 账户管理接口
* 角色管理接口
* 权限资源接口
* 权限查询接口（前端使用）
* 鉴权辅助接口

用于指导：

* 后端 Controller 开发
* 前后端联调
* 接口测试与Mock

---

## 2. 设计原则

* RESTful 风格（资源 + 动作）
* 返回结构统一
* 权限控制基于 `access_resource`
* 接口鉴权统一通过 Spring Security
* 所有写操作具备幂等性设计（结合数据库约束）

---

# 4. 账户管理接口

---

## 4.1 查询账户列表

**GET** `/api/accounts`

### 请求参数

| 参数 | 类型 | 必填 | 说明 | 
| -------- | ------ | -- | --- | 
| username | string | 否 | 用户名 | 
| enabled | int | 否 | 状态 |

---

### 返回示例

```json
{
  "total": 2,
  "list": [
    {
      "id": 1,
      "username": "admin",
      "nickname": "管理员",
      "enabled": 1
    }
  ]
}
```

---

## 4.2 新增账户

**POST** `/api/accounts`

```json
{
  "username": "test",
  "password": "123456",
  "nickname": "测试用户"
}
```

---

## 4.3 修改账户

**PUT** `/api/accounts/{id}`

---

## 4.4 启用/禁用账户

**PATCH** `/api/accounts/{id}/status`

```json
{
  "enabled": 0
}
```

---

## 4.5 绑定角色

**POST** `/api/accounts/{id}/roles`

```json
{
  "roleIds": [
    1,
    2
  ]
}
```

---

## 4.6 解绑角色

**DELETE** `/api/accounts/{id}/roles`

```json
{
  "roleIds": [
    1
  ]
}
```

# 7. 账户权限（直绑）接口

---

## 7.1 设置账户权限（ALLOW / DENY）

**POST** `/api/accounts/{id}/permissions`

```json
{
  "permissions": [
    {
      "permissionId": 10,
      "effectType": "ALLOW"
    },
    {
      "permissionId": 11,
      "effectType": "DENY"
    }
  ]
}
```

---

## 7.2 取消账户权限

**DELETE** `/api/accounts/{id}/permissions`

# 8. 权限查询接口（前端使用）

---

## 8.1 获取当前用户权限集合

**GET** `/api/me/permissions`

---

### 返回示例

```json
{
  "permissions": [
    "sys:account:add",
    "btn:sys:account:add",
    "menu:sys:account",
    "data:sys:account:show-phone"
  ]
}
```

---

## 8.2 获取菜单树

**GET** `/api/me/menus`

---

# 9. 鉴权约定

---

## 9.1 API 权限

后端接口需声明：

```java
@PreAuthorize("hasPermission('sys:account:add')")
```

---

## 9.2 按钮权限

前端根据：

```json
"btn:sys:account:add"
```

控制显示

---

## 9.3 DATA / FILE / CAP

在业务代码中显式判断：

```java
if(hasPermission("data:sys:account:show-phone")){
        // 展示手机号
        }
```
