# 账号与权限数据库详细设计

## 1. 文档目的

本文档用于细化《账号与权限系统设计》中的数据库模型，明确表结构、字段约束、索引设计、状态约定、关联规则与审计字段规范，作为后续建表、DAO 开发、服务实现和测试用例编写的依据。本文档承接系统设计中的核心模型与约束，不重复讨论系统目标与总体架构。

---

## 2. 设计范围

本数据库详细设计覆盖以下六张核心表：

* `sys_account`
* `sys_role`
* `sys_permission`
* `sys_account_role_association`
* `sys_role_permission_association`
* `sys_account_permission_association`

本期不包含：

* 审计流水表
* 登录日志表
* 操作日志表
* 组织架构相关表
* 临时提权相关表

这些内容在系统设计中已被明确排除或放入二期。

---

## 3. 总体设计约束

### 3.1 主键约束

所有核心表与关联表统一使用 `bigint` 作为主键类型，主键字段名统一为 `id`。

### 3.2 审计字段约束

除纯查询视图外，所有表统一包含以下审计字段：

* `created_by`
* `created_time`
* `updated_by`
* `updated_time`

其中：

* `created_by` / `updated_by` 记录操作账户 ID
* 异步任务与定时任务场景下写入预置系统账户 ID，例如 `-1`，与系统设计保持一致。

### 3.3 启停与删除约束

* `sys_account` 不允许逻辑删除，仅允许 `enabled = 0` 禁用。
* `sys_role`、`sys_permission` 允许逻辑删除，使用 `deleted = 1` 表示已删除。
* 三张关联表不做逻辑删除，通过 `enabled = 1/0` 表达当前关系是否生效，与系统设计中的 Upsert 复用机制一致。

### 3.4 权限编码约束

`sys_permission.access_resource`：

* 全局唯一
* 仅允许小写字母、数字、冒号 `:`、短横线 `-`
* 不允许存储通配符 `*`
* 作为权限判定唯一业务编码。

### 3.5 最终权限计算依赖

最终权限按以下顺序计算：

1. 账户级显式拒绝
2. 账户级显式允许
3. 任一启用角色授予
4. 默认拒绝

因此数据库层必须能够稳定表达：

* 账户直绑权限的 `ALLOW / DENY`
* 角色权限只允许 `ALLOW`
* 所有关联关系均可启停。

## 4. 枚举与字段值约定

## 4.1 enabled

适用表：

* `sys_account`
* 三张关联表
* 如有需要也可用于 `sys_role`、`sys_permission`

取值：

* `1`：启用 / 生效
* `0`：禁用 / 不生效

## 4.2 deleted

适用表：

* `sys_role`
* `sys_permission`

取值：

* `0`：未删除
* `1`：已逻辑删除

## 4.3 permission_type

对应 `sys_permission.type`

取值：

* `MENU`
* `BUTTON`
* `API`
* `DATA`
* `FILE`
* `CAPABILITY`

## 4.4 effect_type

仅适用于 `sys_account_permission_association`

取值：

* `ALLOW`
* `DENY`

说明：

* 角色权限表不需要 `effect_type`
* 角色权限天然只表达授权，不表达拒绝，与系统设计一致。

## 5. 表结构详细设计

## 5.1 `sys_account`

### 5.1.1 表用途

存储系统登录账户信息，是认证主体。账户允许禁用，不允许删除。

### 5.1.2 字段设计

| 字段名        | 类型          | 非空 | 默认值               | 说明          |
| ------------ | ------------ | -:   | ----------------- | ----------- | 
| id           | bigint       |  是 | 无                 | 主键          |
| username     | varchar(64)  |  是 | 无                 | 登录名，全局唯一    |
| password     | varchar(255) |  是 | 无                 | bcrypt 哈希密码 |
| nickname     | varchar(64)  |  否 | null              | 显示名称/昵称     |
| real_name    | varchar(64)  |  否 | null              | 真实姓名        |
| phone        | varchar(32)  |  否 | null              | 手机号         |
| email        | varchar(128) |  否 | null              | 邮箱          |
| enabled      | tinyint      |  是 | 1                 | 是否启用        |
| remark       | varchar(500) |  否 | null              | 备注          |
| created_by   | bigint       |  是 | 0                 | 创建人         |
| created_time | datetime     |  是 | CURRENT_TIMESTAMP | 创建时间        |
| updated_by   | bigint       |  是 | 0                 | 更新人         |
| updated_time | datetime     |  是 | CURRENT_TIMESTAMP | 更新时间        |

### 5.1.3 索引设计

* 主键：`pk_sys_account (id)`
* 唯一索引：`uk_sys_account_username (username)`

### 5.1.4 设计说明

* `username` 作为唯一登录名，不允许重复。
* 不设置 `deleted` 字段，避免误删账户。
* `password` 长度需覆盖 bcrypt 哈希串。

---

## 5.2 `sys_role`

### 5.2.1 表用途

存储角色定义，角色仅表达授权集合，不支持继承与拒绝。

### 5.2.2 字段设计

| 字段名          | 类型           | 非空 | 默认值               | 说明                    |
| ------------ | ------------ | -: | ----------------- | --------------------- |
| id           | bigint       |  是 | 无                 | 主键                    |
| role_code    | varchar(64)  |  是 | 无                 | 角色编码，建议全局唯一           |
| role_name    | varchar(64)  |  是 | 无                 | 角色名称                  |
| role_type    | varchar(32)  |  是 | NORMAL            | 角色类型，如 NORMAL / BASIC |
| enabled      | tinyint      |  是 | 1                 | 是否启用                  |
| deleted      | tinyint      |  是 | 0                 | 是否逻辑删除                |
| remark       | varchar(500) |  否 | null              | 备注                    |
| created_by   | bigint       |  是 | 0                 | 创建人                   |
| created_time | datetime     |  是 | CURRENT_TIMESTAMP | 创建时间                  |
| updated_by   | bigint       |  是 | 0                 | 更新人                   |
| updated_time | datetime     |  是 | CURRENT_TIMESTAMP | 更新时间                  |

### 5.2.3 索引设计

* 主键：`pk_sys_role (id)`
* 唯一索引：`uk_sys_role_role_code (role_code)`
* 普通索引：`idx_sys_role_deleted_enabled (deleted, enabled)`

### 5.2.4 设计说明

* `role_type` 用于区分普通角色和基础角色，支持系统设计中“基础角色”的表达。
* 不设计 `parent_id`，因为本系统不采用角色继承。

---

## 5.3 `sys_permission`

### 5.3.1 表用途

存储权限资源定义，是统一资源集合与权限编码承载表。

### 5.3.2 字段设计

| 字段名             | 类型           | 非空 | 默认值               | 说明               |
| --------------- | ------------ | -: | ----------------- | ---------------- |
| id              | bigint       |  是 | 无                 | 主键               |
| permission_name | varchar(128) |  是 | 无                 | 权限名称             |
| type            | varchar(32)  |  是 | 无                 | 权限类型             |
| access_resource | varchar(128) |  是 | 无                 | 权限唯一编码           |
| business_group  | varchar(64)  |  是 | 无                 | 业务分组             |
| parent_id       | bigint       |  否 | null              | 父节点 ID，仅用于树展示    |
| sort_no         | int          |  是 | 0                 | 排序号              |
| enabled         | tinyint      |  是 | 1                 | 是否启用             |
| deleted         | tinyint      |  是 | 0                 | 是否逻辑删除           |
| remark          | varchar(500) |  否 | null              | 备注               |
| created_by      | bigint       |  是 | 0                 | 创建人              |
| created_time    | datetime     |  是 | CURRENT_TIMESTAMP | 创建时间             |
| updated_by      | bigint       |  是 | 0                 | 更新人              |
| updated_time    | datetime     |  是 | CURRENT_TIMESTAMP | 更新时间             |

### 5.3.3 索引设计

* 主键：`pk_sys_permission (id)`
* 唯一索引：`uk_sys_permission_access_resource (access_resource)`
* 普通索引：`idx_sys_permission_parent_id (parent_id)`
* 普通索引：`idx_sys_permission_group_type (business_group, type)`
* 普通索引：`idx_sys_permission_deleted_enabled (deleted, enabled)`

### 5.3.4 设计说明

* `access_resource` 为业务唯一权限编码，判定权限时以此为准。
* `parent_id` 仅用于前端展示和授权树构建，不参与权限推导。
* `BUTTON`、`API`、`DATA`、`FILE`、`CAPABILITY` 类型不强依赖 UI 字段。

## 5.4 `sys_account_role_association`

### 5.4.1 表用途

存储账户与角色的绑定关系，支持启停复用与幂等绑定解绑。

### 5.4.2 字段设计

| 字段名          | 类型           | 非空 | 默认值               | 说明       |
| ------------ | ------------ | -: | ----------------- | -------- |
| id           | bigint       |  是 | 无                 | 主键       |
| account_id   | bigint       |  是 | 无                 | 账户 ID    |
| role_id      | bigint       |  是 | 无                 | 角色 ID    |
| enabled      | tinyint      |  是 | 1                 | 当前关系是否生效 |
| remark       | varchar(500) |  否 | null              | 备注       |
| created_by   | bigint       |  是 | 0                 | 创建人      |
| created_time | datetime     |  是 | CURRENT_TIMESTAMP | 创建时间     |
| updated_by   | bigint       |  是 | 0                 | 更新人      |
| updated_time | datetime     |  是 | CURRENT_TIMESTAMP | 更新时间     |

### 5.4.3 索引设计

* 主键：`pk_sys_account_role_association (id)`
* 唯一索引：`uk_sys_account_role (account_id, role_id)`
* 普通索引：`idx_sys_account_role_account_enabled (account_id, enabled)`

### 5.4.4 设计说明

| enabled       | tinyint      |  是 | 1                 | 当前关系是否生效 |
| remark        | varchar(500) |  否 | null              | 备注       |
| created_by    | bigint       |  是 | 0                 | 创建人      |

## 5.6 `sys_account_permission_association`

### 5.6.1 表用途

存储账户直绑权限关系，用于表达账户级显式授权与显式拒绝。

### 5.6.2 字段设计

| 字段名           | 类型           | 非空 | 默认值               | 说明           |
| ------------- | ------------ | -: | ----------------- | ------------ |
| id            | bigint       |  是 | 无                 | 主键           |
| account_id    | bigint       |  是 | 无                 | 账户 ID        |
| permission_id | bigint       |  是 | 无                 | 权限 ID        |
| effect_type   | varchar(16)  |  是 | 无                 | ALLOW / DENY |
| enabled       | tinyint      |  是 | 1                 | 当前关系是否生效     |
| remark        | varchar(500) |  否 | null              | 备注           |
| created_by    | bigint       |  是 | 0                 | 创建人          |
| created_time  | datetime     |  是 | CURRENT_TIMESTAMP | 创建时间         |
| updated_by    | bigint       |  是 | 0                 | 更新人          |
| updated_time  | datetime     |  是 | CURRENT_TIMESTAMP | 更新时间         |

### 5.6.3 索引设计

* 主键：`pk_sys_account_permission_association (id)`
* 唯一索引：`uk_sys_account_permission (account_id, permission_id)`
* 普通索引：`idx_sys_account_permission_account_enabled (account_id, enabled)`
* 普通索引：`idx_sys_account_permission_permission_enabled (permission_id, enabled)`
* 普通索引：`idx_sys_account_permission_effect_type (effect_type)`

## 6. 表关系说明

### 6.1 关系总览

* `sys_account` 与 `sys_role`：多对多，通过 `sys_account_role_association`
* `sys_role` 与 `sys_permission`：多对多，通过 `sys_role_permission_association`
* `sys_account` 与 `sys_permission`：多对多，通过 `sys_account_permission_association`

### 6.2 外键策略

建议：

* **逻辑上有关联，物理上不强制外键约束**
* 由应用层和事务保障数据一致性

原因：

* 便于批量导入与历史修复
* 降低复杂更新与逻辑删除时的约束干扰
* 更符合多数业务系统实践

如果团队偏好强约束，也可在开发库开启外键验证，但生产推荐以应用校验为主。

## 7. 关键查询与实现约束

## 7.1 查询账户最终权限所需数据

计算账户最终权限时，至少需要查询：

1. 启用的账户角色关系
2. 启用、未删除的角色
3. 启用的角色权限关系
4. 启用、未删除的权限
5. 启用的账户权限关系
6. 启用中的账户自身状态

## 7.2 查询条件建议

在涉及角色与权限关联查询时，统一增加以下过滤：

* 主体表：

    * `enabled = 1`
    * `deleted = 0`（如表支持）
* 关联表：

    * `enabled = 1`

避免禁用或逻辑删除数据参与权限计算。

---

## 8. 逻辑删除与级联处理规范

## 8.1 删除角色

当 `sys_role.deleted` 更新为 `1` 时，需要同步处理：

* `sys_account_role_association.enabled = 0`
* `sys_role_permission_association.enabled = 0`

## 8.2 删除权限

当 `sys_permission.deleted` 更新为 `1` 时，需要同步处理：

* `sys_role_permission_association.enabled = 0`
* `sys_account_permission_association.enabled = 0`

## 8.3 禁用账户

当 `sys_account.enabled` 更新为 `0` 时：

* 不需要物理移除其角色与权限关联
* 认证与鉴权阶段直接阻断其登录和请求访问

---

## 9. 幂等与 Upsert 约定

## 9.1 绑定

* 若关联记录不存在，则插入新记录并 `enabled = 1`。
* 若关联记录存在且 `enabled = 0`，则更新为 `enabled = 1`。
* 若关联记录存在且 `enabled = 1`，则视为成功，不报错。

## 9.2 解绑

* 若关联记录不存在，则视为成功，不报错。
* 若关联记录存在，则更新为 `enabled = 0`。

## 9.3 账户权限关系修改

若 `sys_account_permission_association` 已存在：

* 允许修改 `effect_type`
* 同时刷新 `enabled`、`updated_by`、`updated_time`

适用于：

* 从 `ALLOW` 改为 `DENY`
* 从 `DENY` 改为 `ALLOW`

---

## 10. 初始化数据建议

## 10.1 系统保留账户

建议初始化：

* `id = -1`
* `username = SYSTEM_JOB`
* `enabled = 0` 或不可登录态

用途：

* 异步任务
* 定时任务
* 消费者处理上下文审计写入

## 10.2 基础角色

建议初始化若干 `role_type = BASIC` 的基础角色，用于承载通用权限。

## 10.3 权限初始化

建议预置基础模块权限：

* 菜单权限
* 按钮权限
* API 权限
* 少量演示型 DATA / FILE / CAPABILITY 权限

---

## 11. 建表 SQL 风格建议

建议统一：

* 字符集：`utf8mb4`
* 排序规则：`utf8mb4_unicode_ci` 或团队标准
* 存储引擎：`InnoDB`
* 时间字段统一使用 `datetime`
* 审计字段默认值与更新时间策略统一
